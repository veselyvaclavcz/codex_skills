# Layout

Use this reference when you need to understand or explain the shared project-memory structure.

## Root

The skill stores shared context in:

`docs/project-memory/`

This location is intentionally:

- visible in the repo
- easy to diff
- easy for fresh threads to discover
- separate from product docs that may be user-facing

The preferred automation entry point is `scripts/sync_project_memory.py`, which bootstraps this layout automatically if it is missing.

The skill also maintains a managed block in the project `AGENTS.md` to reinforce two repository-level habits:

- use shared project memory by default
- use model orchestration only when delegation is worth the cost and complexity

## Top-Level Files

### `README.md`

Quick orientation for fresh threads.

### `project-summary.md`

Longer-lived explanation of what the project is, what it does, and what constraints matter.

### `current-state.md`

Short current snapshot of implementation status, in-flight work, and recent state.

### `decisions.md`

Durable technical or product decisions that should outlive one thread.

### `open-questions.md`

Issues that are unresolved and likely to matter later.

### `conventions.md`

Working rules, conventions, naming rules, testing expectations, release rules, or other repo-local norms.

## Conversation Notes

Conversation notes live under:

`docs/project-memory/conversations/YYYY/`

Each note is one markdown file with frontmatter-like metadata and sections:

- Summary
- Decisions
- Open Questions
- Impacted Files
- Next Steps

Use one note per substantial thread milestone, not one note per tiny message.

## Index

`docs/project-memory/conversations/index.md`

This file is generated by `rebuild_conversation_index.py` and gives later threads an easy entry point.
