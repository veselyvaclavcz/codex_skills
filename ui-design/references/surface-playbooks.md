# Surface Playbooks

Use the matching section for the interface you are designing.

## Dashboard / Admin / Data-heavy App

- Prefer full-width layouts.
- Use compact spacing and smaller type where it improves scan speed.
- Keep filters and time controls close to the chart or table they affect.
- Give charts explicit axis labels, legends only when needed, and visible time windows.
- Replace decorative cards with meaningful summaries.
- Put logs, status, and configuration into clear secondary zones such as tabs, accordions, or lower sections.
- Avoid giant hero sections inside operational tools.

## Forms / Settings / CRUD Screens

- Keep one obvious primary action per section.
- Use persistent labels, not placeholder-only inputs.
- Group fields by user intent, not by database shape.
- Put inline help next to the field, not buried below the page.
- Make validation immediate and specific.
- If a field is optional, the UI should make that clear without clutter.

## Landing Pages / Marketing Pages

- Lead with one strong claim and one clear next step.
- Build section rhythm with contrast in scale, layout, and pacing.
- Use proof early: product image, metrics, testimonial, recognizable logos, or outcome.
- Avoid empty "feature grids" unless each item says something distinct.
- Decorative treatment should support the story, not replace it.

## Mobile / Small Surfaces

- One dominant job per screen.
- Respect thumb reach for primary actions.
- Collapse low-priority controls behind sheets, drawers, tabs, or segmented views.
- Keep section headers short and action-oriented.
- Test vertical rhythm carefully; mobile UIs break fast when spacing is inconsistent.

## Desktop / Native-like Productivity UI

- Use denser layouts than marketing pages.
- Make side panels, inspectors, and keyboard-friendly actions feel stable.
- Surface selection state, mode state, and unsaved changes clearly.
- Use restrained color. Productivity tools benefit more from hierarchy than from decoration.

## Shared Quality Checks

- The user should understand the screen's job in under 5 seconds.
- The primary action should be obvious without reading every label.
- Empty and loading states should look intentional.
- Text should sound like product language, not placeholder filler.
- If the screen is busy, remove weak containers before shrinking everything else.
