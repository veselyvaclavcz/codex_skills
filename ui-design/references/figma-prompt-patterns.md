# Figma Prompt Patterns

Use these as prompt shapes, not fixed magic phrases.

They are meant for editable generation inside Figma, followed by manual refinement.

## Seed prompts

Use these when starting from a blank canvas:

- `Create a compact CRM review screen for sales reps. Prioritize scanability, next actions, and dense but readable tables.`
- `Create a dashboard for monitoring warehouse operations. Make the hierarchy operational and calm, not flashy.`
- `Create a pricing page for a B2B developer tool. Keep the structure simple, the messaging direct, and the typography confident.`

## Structural mutations

Use these when the overall vibe is fine, but the layout is wrong:

- `Keep the design direction, but change the navigation to a left sidebar and rebalance the page around it.`
- `Keep the same visual language, but simplify this to a single-column structure with clearer section order.`
- `Keep the theme and typography, but make the page denser and more operational.`

## Stylistic mutations

Use these when the structure is fine, but the tone is off:

- `Keep the layout, but make the design feel more technical and less generic.`
- `Keep the structure, but make the typography more editorial and the chrome more restrained.`
- `Keep the content structure, but simplify the visual language and remove decorative styling.`

## Simplification prompts

Use these after an AI draft feels busy or over-designed:

- `Simplify this design direction. Remove visual noise, reduce unnecessary boxes, and make hierarchy clearer.`
- `Keep the same content, but reduce decoration and make the page feel more deliberate.`
- `Tighten spacing, reduce secondary emphasis, and make the primary action clearer.`

## Section-level prompts

Use selection-scoped prompts when only part of the design needs to change:

- `Apply this change only to the selected section: move the filters above the table and compress the control bar.`
- `Apply this change only to the selected hero: keep the copy, but rebuild the composition around stronger typography.`
- `Apply this change only to the selected cards: reduce repetition and make them read as one system.`

## Prompting rules

- mention the target user and their job
- mention the dominant constraint such as density, mobile space, or readability
- specify whether the change is structural, stylistic, or simplifying
- keep one prompt focused on one kind of change

## Stop-prompting rule

If the remaining issues are mostly spacing, type size, alignment, or emphasis, stop prompting and edit manually in Figma.
