# Page Type Variants

Use this file after `catalog.md` when the work benefits from several ready-made design directions.

Choose one anchor direction. Use a second direction only for a narrow concern such as typography, density, or navigation.

## Dashboard / Admin / Data App

- `dashboard`: cloud-platform, strong hierarchy, dark operational UI
- `application`: top-bar app shell, developer-tool feel, lighter product UI
- `shadcn`: restrained monochrome product UI, good for code-heavy tools
- `enterprise`: structured workflow surfaces, calmer business feel
- `mono`: technical, utilitarian, useful for internal tools
- `clean`: simple, broad default when the product needs polish without drama

## Landing Page / Marketing Site

- `premium`: polished consumer-tech marketing
- `refined`: upscale but restrained brand presentation
- `editorial`: content-led storytelling with typographic authority
- `modern`: broadly contemporary marketing without extreme personality
- `bold`: assertive, louder hierarchy for launches and campaigns
- `minimal`: stripped-back composition with whitespace and calm tone

## Form / Settings / Auth / Onboarding

- `clean`: safe default for clear form-heavy experiences
- `shadcn`: compact, practical form surfaces
- `application`: good for authenticated app flows
- `minimal`: lighter, calmer settings and account surfaces
- `simple`: low-chrome patterns where clarity matters more than personality
- `enterprise`: good when the form belongs to a business workflow

## Documentation / Blog / Knowledge Surface

- `editorial`: long-form reading and feature stories
- `publication`: stronger publication feel with article-first hierarchy
- `paper`: quiet reading surfaces and print-like rhythm
- `minimal`: restrained docs with space and clarity
- `mono`: technical writing, release notes, changelogs, code-adjacent docs

## Ecommerce / Product Catalog / Offer Page

- `professional`: trustworthy structured commerce surfaces
- `premium`: premium-product merchandising and hero composition
- `clean`: broad ecommerce polish without over-branding
- `elegant`: softer upscale merchandising
- `refined`: premium retail or boutique positioning

## Creative / Brand-Heavy / Playful Experience

- `artistic`: expressive visual direction with stronger form language
- `creative`: broad creative-brand mode when the brief wants novelty
- `doodle`: hand-drawn or playful personality
- `retro`: nostalgic but more general than game-specific themes
- `pacman`: very specific playful arcade voice
- `vibrant`: saturated and energetic brand work

## Futuristic / Experimental / Thematic

- `futuristic`: speculative product feel, use sparingly
- `cosmic`: atmospheric and thematic, useful for campaign pages
- `neon`: louder night-mode energy, only when the brief supports it
- `perspective`: dramatic depth and visual motion cues
- `glassmorphism`: use only when translucency is actually part of the brand
- `gradient`: useful for hero-led marketing, rarely for product UI

## Dense Utility / Internal Ops / Serious B2B

- `dashboard`: dark high-density default
- `enterprise`: business-system structure and workflow clarity
- `shadcn`: minimal and code-adjacent
- `mono`: sober technical utility
- `flat`: performance-first and low-decoration
- `corporate`: more traditional business tone when the product should feel conventional

## Selection Heuristics

- If the project already has a visual system, choose the closest variant only as inspiration.
- If the user asks for concepts, pick 2-4 variants from the same section and explain the trade-offs.
- If the requested variant would obviously fight the information architecture, reject it and choose a better anchor.
- For internal tools, prefer `dashboard`, `application`, `shadcn`, `enterprise`, `mono`, `clean`, or `flat` before more theatrical styles.
